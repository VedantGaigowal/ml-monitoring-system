# ML Model Monitoring & Drift Detection System


A system that watches a machine learning model in production — detecting when it starts going wrong, alerting you about it, and showing the recovery after retraining. Built with scikit-learn and Streamlit.

---

## What It Does

You train a model. You deploy it. Months later the data it receives looks nothing like what it learned on. Predictions quietly get worse. Nobody notices until the damage is done.

This project catches that happening.

It monitors the gap between training data and production data, tracks how predictions shift over time, measures model performance as ground truth arrives, and fires alerts when things fall below acceptable levels. When you retrain on fresh data, the dashboard shows the recovery side by side with the original model's degradation.

---

## The Setup

A fictional company over 15 months. You are the data scientist predicting which employees are likely to quit.

| Period | Situation | Model Accuracy |
|---|---|---|
| Jan 2023 | Healthy company — this is where training data comes from | 96% |
| Jun 2023 | Salary freeze, overwork begins, data starts shifting | 67% |
| Dec 2023 | Layoffs, extreme hours, distributions nothing like training | 60% |
| Mar 2024 | Recovery — retrain here | 97% |

The original model drops from 96% to 60% as conditions change. The retrained model on recovery data hits 97%. That arc is what the whole project demonstrates.

## Stack

- **scikit-learn** — Random Forest, metrics
- **scipy** — KS test, Chi-Square test
- **pandas / numpy** — data handling
- **matplotlib** — charts
- **streamlit** — dashboard

## Project Structure

```
ml-monitoring-system/
│
├── scripts/
│   ├── train_model.py              ← generates data + trains the model
│   ├── feature_pipeline.py         ← simulates drifted production data
│   ├── drift_detection.py          ← statistical tests on every feature
│   ├── prediction_monitoring.py    ← runs inference, logs predictions
│   ├── performance_monitoring.py   ← accuracy / precision / recall / F1 + alerts
│   ├── simulate_timeline.py        ← creates all 4 time-period datasets
│   └── retrain_model.py            ← train → watch it fail → retrain → compare
│
├── dashboard/
│   └── app.py                      ← 6-page Streamlit dashboard
│
├── data/                           ← generated by running the scripts
├── models/                         ← generated by running the scripts
└── requirements.txt
```

## Running It

```bash
git clone https://github.com/yourusername/ml-monitoring-system.git
cd ml-monitoring-system
pip install -r requirements.txt
```

Run the scripts in order:

```bash
python scripts/train_model.py
python scripts/feature_pipeline.py
python scripts/prediction_monitoring.py
python scripts/simulate_timeline.py
python scripts/retrain_model.py

streamlit run dashboard/app.py
```

Open **http://localhost:8501**

---

## The Dashboard

**Model Overview** — accuracy, model health status, feature importances

**Feature Drift** — which features have shifted and by how much, with distribution charts

**Prediction Monitoring** — what the model is predicting over time, confidence scores, rolling attrition rate

**Performance Monitoring** — accuracy, precision, recall, F1, confusion matrix

**Alerts** — everything that needs attention in one place

**Timeline and Retraining** — both models plotted across all four periods so you can see exactly where things went wrong and where retraining fixed it

## Troubleshooting

| Problem | Fix |
|---|---|
| Dashboard page 6 is empty | Run `simulate_timeline.py` and `retrain_model.py` first |
| FileNotFoundError on prediction_logs.csv | Run `prediction_monitoring.py` before launching dashboard |
| No drift showing up | Make sure `drift_factor=1.4` in `feature_pipeline.py` |
| ModuleNotFoundError | Run `pip install -r requirements.txt` |
